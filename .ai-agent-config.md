```markdown
## 3. Git Workflow (MANDATORY)
After EVERY meaningful change:
- `git add <file_name>`
- Write clear, scoped commit message.
- Avoid atomic micro-commits.

- Each commit must represent a logical unit of work.

Example:
```

git add src/modules/attendance/attendance.service.ts
git commit -m "feat(attendance): implement multi-tenant daily attendance logic with validation"
